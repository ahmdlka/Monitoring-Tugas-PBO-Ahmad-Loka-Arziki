Tugas 3: Implementasi Clock dengan GUI dan Number Display

16 September 2025
Waktu pengerjaan: 2 hari
Class Object Java GUI Swing

Deskripsi Tugas

Pada tugas ketiga mata kuliah Pemrograman Berbasis Objek, saya mempelajari konsep composition, encapsulation, dan penggunaan GUI dengan Swing di Java. Tugas ini mengharuskan saya untuk membuat program simulasi jam digital menggunakan BlueJ, yang terdiri dari tiga class utama: Clock untuk antarmuka grafis, ClockDisplay untuk logika tampilan waktu, dan NumberDisplay untuk perhitungan angka dengan limit rollover. Program ini mendemonstrasikan bagaimana class saling bergantung untuk membentuk sistem lengkap, dengan interaksi melalui method dan object instantiation.

Implementasi Program

Saya mengimplementasikan program Clock dengan tiga class yang saling terkait. Class Clock membuat GUI menggunakan JFrame dan Swing components, yang memanggil method dari ClockDisplay untuk update waktu. Class ClockDisplay menggunakan dua instance NumberDisplay (untuk hours dan minutes) untuk menghitung dan memformat waktu dalam string "HH:MM". Class NumberDisplay menangani operasi dasar seperti increment dengan rollover. Berikut adalah kode lengkap untuk masing-masing class:

Class Clock

import java.awt.*;
import java.awt.event.*;
import javax.swing.*;
import javax.swing.border.*;

public class Clock {
    private JFrame frame;
    private JLabel label;
    private ClockDisplay clock;
    private boolean clockRunning = false;
    private TimerThread timerThread;

    public Clock() {
        makeFrame();
        clock = new ClockDisplay();
    }

    private void start() {
        if (!clockRunning) {
            clockRunning = true;
            timerThread = new TimerThread();
            timerThread.start();
        }
    }

    private void stop() {
        clockRunning = false;
    }

    private void step() {
        clock.timeTick();
        label.setText(clock.getTime());
    }

    private void showAbout() {
        JOptionPane.showMessageDialog(frame,
            "Clock Version 1.0\nA simple interface for the clock display project",
            "About Clock", JOptionPane.INFORMATION_MESSAGE);
    }

    private void quit() {
        System.exit(0);
    }

    private void makeFrame() {
        frame = new JFrame("Clock");
        JPanel contentPane = (JPanel) frame.getContentPane();
        contentPane.setBorder(new EmptyBorder(1, 60, 1, 60));
        contentPane.setLayout(new BorderLayout(12, 12));
        
        label = new JLabel("00:00", SwingConstants.CENTER);
        Font displayFont = label.getFont().deriveFont(96.0f);
        label.setFont(displayFont);
        contentPane.add(label, BorderLayout.CENTER);
        
        JPanel toolbar = new JPanel();
        toolbar.setLayout(new GridLayout(1, 0));
        JButton startButton = new JButton("Start");
        startButton.addActionListener(e -> start());
        toolbar.add(startButton);
        
        JButton stopButton = new JButton("Stop");
        stopButton.addActionListener(e -> stop());
        toolbar.add(stopButton);
        
        JButton stepButton = new JButton("Step");
        stepButton.addActionListener(e -> step());
        toolbar.add(stepButton);
        JPanel flow = new JPanel();

        flow.add(toolbar);
        contentPane.add(flow, BorderLayout.SOUTH);

        frame.pack();
        Dimension d = Toolkit.getDefaultToolkit().getScreenSize();
        frame.setLocation(d.width / 2 - frame.getWidth() / 2, d.height / 2 - frame.getHeight() / 2);
        frame.setVisible(true);

        makeMenuBar(frame);

    }
    
    private void makeMenuBar(JFrame frame) {

        final int SHORTCUT_MASK = Toolkit.getDefaultToolkit().getMenuShortcutKeyMask();
        JMenuBar menubar = new JMenuBar();
        frame.setJMenuBar(menubar);
        JMenu menu;
        JMenuItem item;


        menu = new JMenu("File");

        menubar.add(menu);


        item = new JMenuItem("About Clock...");

        item.addActionListener(e -> showAbout());

        menu.add(item);


        menu.addSeparator();


        item = new JMenuItem("Quit");

        item.setAccelerator(KeyStroke.getKeyStroke(KeyEvent.VK_Q, SHORTCUT_MASK));

        item.addActionListener(e -> quit());

        menu.add(item);

    }
    
    class TimerThread extends Thread {
        public void run() {
            while (clockRunning) {

                step();

                pause();

            }
        }
        private void pause() {
            try {

                Thread.sleep(1000);

            } catch (InterruptedException exc) {}
        }
    }
    
    public static void main(String[] args) {
        System.out.println("Hello World");
        SwingUtilities.invokeLater(() -> new Clock());

    }
}

Class ClockDisplay

public class ClockDisplay{
    private NumberDisplay hours;
    private NumberDisplay minutes;
    private String displayString; // simulates the actual display
    /**
    * Constructor for ClockDisplay objects. This constructor
    * creates a new clock set at 00:00.
    */
    public ClockDisplay()
    {
        hours = new NumberDisplay(24);
        minutes = new NumberDisplay(60);
        updateDisplay();
    }
    /**
    * Constructor for ClockDisplay objects. This constructor
    * creates a new clock set at the time specified by the
    * parameters.
    */
    public ClockDisplay(int hour, int minute)
    {
        hours = new NumberDisplay(24);
        minutes = new NumberDisplay(60);
        setTime(hour, minute);
    }
    /**
    * This method should get called once every minute - it
    * makes the clock display go one minute forward.
    */
    public void timeTick()
    {
        minutes.increment();
            if(minutes.getValue() == 0) { // it just rolled over!
            hours.increment();
        }
        updateDisplay();
    }
    /**
    * Set the time of the display to the specified hour and
    * minute.
    */
    public void setTime(int hour, int minute)
    {
        hours.setValue(hour);
        minutes.setValue(minute);
        updateDisplay();
    }
    /**
    * Return the current time of this display in the format
    * HH:MM.
    */
    public String getTime()
    {
        return displayString;
    }
    /**
    * Update the internal string that represents the
    * display.
    */
    private void updateDisplay()
    {
        displayString = hours.getDisplayValue() + ":" 
        + minutes.getDisplayValue();
    }
}

Class NumberDisplay

public class NumberDisplay
{
    private int limit;
    private int value;
    /**
    * Constructor for objects of class NumberDisplay
    */
    public NumberDisplay(int rollOverLimit)
    {
        limit = rollOverLimit;
        value = 0;
    }
    /**
    * Return the current value.
    */
    public int getValue()
    {
        return value;
    }
    /**
    * Set the value of the display to the new specified
    * value. If the new value is less than zero or over the
    * limit, do nothing.
    */
    public void setValue(int replacementValue)
    {
        if((replacementValue >= 0) &&
        (replacementValue < limit)) {
            value = replacementValue;
        }
    }
    /**
    * Return the display value (that is, the current value
    * as a two-digit String. If the value is less than ten,
    * it will be padded with a leading zero).
    */
    public String getDisplayValue()
    {
        if(value < 10) {
            return "0" + value;
        }
        else {
            return "" + value;
        }
    }
    /**
    * Increment the display value by one, rolling over to zero if
    * the limit is reached.
    */
    public void increment()
    {
        value = (value + 1) % limit;
    }
}

Penjelasan Konsep

Program Clock ini mendemonstrasikan konsep dasar Pemrograman Berbasis Objek seperti composition (class ClockDisplay menggabungkan dua NumberDisplay), encapsulation (atribut privat diakses via method), dan modularitas (NumberDisplay reusable untuk jam/menit). Berikut penjelasan garis besar setiap method pada ketiga class:

Class Clock

Class Clock merupakan class utama yang bertanggung jawab untuk membuat antarmuka grafis (GUI) menggunakan Swing di Java. Class ini mengintegrasikan class ClockDisplay untuk menampilkan waktu dan mengelola interaksi pengguna melalui tombol dan menu. Ia menggunakan konsep encapsulation untuk menyembunyikan detail GUI di balik method privat, serta composition dengan membuat instance dari ClockDisplay dan thread untuk simulasi waktu berjalan.

  • Constructor Clock() → Menginisialisasi GUI dengan memanggil makeFrame() untuk membuat jendela utama (JFrame) berjudul "Clock". Ia juga membuat instance baru dari ClockDisplay untuk mengelola logika waktu. Frame ditempatkan di tengah layar dan dibuat visible. Ini adalah entry point utama untuk program, yang dipanggil dari main().
  • Method start() (privat) → Memulai jam berjalan secara otomatis. Jika jam belum berjalan (clockRunning false), ia mengatur flag menjadi true dan membuat serta menjalankan thread baru (TimerThread) yang akan memanggil step() setiap detik. Ini mensimulasikan jam real-time.
  • Method stop() (privat) → Menghentikan jam dengan mengatur flag clockRunning menjadi false. Thread akan berhenti setelah loop selesai, sehingga waktu tidak lagi bertambah otomatis.
  • Method step() (privat) → Melangkahkan waktu satu kali secara manual. Ia memanggil timeTick() pada instance ClockDisplay untuk menambah satu menit, kemudian memperbarui label tampilan dengan string waktu.
  • Method getTime() → Digunakan untuk mode step-by-step atau oleh thread.
  • Method showAbout() (privat) → Menampilkan dialog informasi (JOptionPane) tentang program, termasuk versi dan deskripsi singkat. Dipanggil dari menu "About Clock...".
  • Method quit() (privat) → Menutup program dengan memanggil System.exit(0). Dipanggil dari menu "Quit" dengan shortcut keyboard (Ctrl+Q atau Cmd+Q).
  • Method makeFrame() (privat) → Membangun elemen GUI utama. Membuat JFrame, panel konten dengan border kosong, label tengah untuk menampilkan waktu (dengan font besar 96pt), dan toolbar bawah dengan tombol Start, Stop, dan Step. Tombol-tombol ini terhubung ke event listener (ActionListener) yang memanggil method terkait. Frame di-pack dan diposisikan di tengah layar, serta menu bar dibuat dengan makeMenuBar().
  • Method makeMenuBar(JFrame frame) (privat) → Membuat menu bar atas dengan submenu "File" yang berisi item "About Clock..." (memanggil showAbout()) dan "Quit" (dengan accelerator key). Menggunakan shortcut mask sistem untuk kompatibilitas cross-platform.
  • Inner Class TimerThread extends Thread → Thread khusus untuk menjalankan jam secara periodik. Method run() berloop selama clockRunning true, memanggil step() untuk update waktu, dan pause() untuk tidur 1 detik (menggunakan Thread.sleep(1000)). Ini menangani concurrency agar GUI tidak freeze saat jam berjalan.
  • Method main(String[] args) (static) → Entry point program. Mencetak "Hello World" ke console (untuk testing), kemudian menjalankan Clock() di event dispatch thread Swing (SwingUtilities.invokeLater()) untuk memastikan GUI thread-safe.

Class ClockDisplay

Class ClockDisplay mengelola logika tampilan waktu dalam format HH:MM, menggunakan dua instance dari NumberDisplay (satu untuk jam dengan limit 24, satu untuk menit dengan limit 60). Ini adalah class inti untuk simulasi jam digital, yang menerapkan konsep composition (menggabungkan NumberDisplay) dan encapsulation (menyembunyikan detail perhitungan di balik method publik). Class ini dipanggil oleh Clock untuk mendapatkan string waktu dan men-update state.

  • Constructor ClockDisplay() → Constructor default yang membuat jam baru dimulai dari 00:00. Membuat instance NumberDisplay untuk hours (limit 24) dan minutes (limit 60), kemudian memanggil updateDisplay() untuk menginisialisasi string tampilan awal.
  • Constructor ClockDisplay(int hour, int minute) → Constructor overload yang membuat jam dengan waktu spesifik. Seperti constructor default, ia membuat instance NumberDisplay, tapi langsung memanggil setTime(hour, minute) untuk mengatur nilai awal hours dan minutes, diikuti updateDisplay().
  • Method timeTick() (publik) → Menambah satu menit ke jam, mensimulasikan tick setiap menit. Memanggil increment() pada minutes; jika minutes mencapai 0 (rollover), maka increment() juga dipanggil pada hours. Akhirnya, updateDisplay() dipanggil untuk refresh string tampilan. Ini adalah method kunci yang dipanggil oleh Clock untuk update waktu.
  • Method setTime(int hour, int minute) (publik) → Mengatur waktu jam ke nilai spesifik. Memanggil setValue() pada hours dan minutes dengan parameter yang diberikan (hanya jika valid, i.e., dalam limit). Kemudian, updateDisplay() untuk memperbarui string. Berguna untuk reset atau inisialisasi waktu manual.
  • Method getTime() (publik) → Mengembalikan string waktu saat ini dalam format "HH:MM" (dari displayString). Ini dipanggil oleh Clock untuk update label GUI, memastikan tampilan selalu sinkron dengan state internal.
  • Method updateDisplay() (privat) → Memperbarui string internal displayString dengan menggabungkan getDisplayValue() dari hours dan minutes, dipisahkan oleh tanda ":" (misalnya, "12:05"). Dipanggil setiap kali state berubah untuk menjaga konsistensi tampilan.

Class NumberDisplay

Class NumberDisplay adalah class dasar (foundation) untuk menangani tampilan dan perhitungan angka tunggal dengan limit rollover (misalnya, 0-23 untuk jam atau 0-59 untuk menit). Ia menerapkan encapsulation dengan menyembunyikan variabel limit dan value di balik method, serta konsep modularitas agar bisa digunakan ulang di ClockDisplay. Class ini fokus pada operasi sederhana seperti increment dan format string dua digit.

  • Constructor NumberDisplay(int rollOverLimit) → Menginisialisasi display dengan batas maksimum (limit) yang diberikan, seperti 24 untuk jam atau 60 untuk menit. Mengatur value awal menjadi 0. Ini memungkinkan fleksibilitas saat dibuat instance di ClockDisplay.
  • Method getValue() (publik) → Mengembalikan nilai integer saat ini (value). Berguna untuk membaca state internal tanpa memodifikasinya, misalnya untuk debugging atau validasi di class lain.
  • Method setValue(int replacementValue) (publik) → Mengatur nilai baru ke value, tapi hanya jika nilai tersebut valid (antara 0 dan limit-1). Jika tidak valid (negatif atau melebihi limit), tidak ada perubahan. Ini mencegah state tidak konsisten, dipanggil oleh ClockDisplay saat set waktu.
  • Method getDisplayValue() (publik) → Mengembalikan nilai sebagai string dua digit (HH atau MM). Jika value < 10, tambahkan "0" di depan (padding, misalnya "05"); jika >=10, konversi langsung ke string (misalnya "12"). Ini memastikan format tampilan konsisten untuk GUI, dipanggil oleh ClockDisplay untuk build string waktu.
  • Method increment() (publik) → Menambah value sebanyak 1, dengan rollover ke 0 jika mencapai limit (menggunakan modulo % limit). Misalnya, untuk menit: 59 +1 menjadi 0. Ini adalah operasi dasar untuk simulasi waktu berjalan, dipanggil oleh ClockDisplay di timeTick().

Kesimpulan

Tugas ini memperkuat pemahaman tentang composition dan encapsulation dalam PBO. Class Clock mengintegrasikan GUI dengan logika inti dari ClockDisplay, yang bergantung pada NumberDisplay untuk perhitungan modular. Implementasi ini menunjukkan bagaimana object saling berinteraksi untuk membentuk aplikasi lengkap, termasuk penggunaan thread untuk simulasi real-time. Konsep ini menjadi dasar untuk proyek PBO lebih kompleks seperti inheritance atau polymorphism di tugas selanjutnya.
