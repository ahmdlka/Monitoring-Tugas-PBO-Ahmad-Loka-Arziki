Tugas 4: Implementasi Aplikasi Formulir Rencana Studi (FRS) dengan Java

13 Oktober 2025
Waktu pengerjaan: 3 hari

Deskripsi Tugas

Tugas keempat mata kuliah Pemrograman Berbasis Objek ini berfokus pada pengembangan aplikasi Formulir Rencana Studi (FRS) berbasis konsol menggunakan Java. Aplikasi ini dirancang untuk mensimulasikan sistem pengelolaan rencana studi mahasiswa di lingkungan akademik, dengan fitur utama seperti login mahasiswa, melihat daftar kelas, mengambil kelas, menghapus kelas, dan menampilkan FRS. Program ini terdiri dari lima kelas utama: Mahasiswa, Dosen, Kelas, Matakuliah, dan DisplayFRS, serta kelas Main sebagai antarmuka konsol. Tugas ini menekankan penerapan konsep PBO seperti encapsulation, composition, dan penggunaan struktur data ArrayList untuk mengelola koleksi data.

Aplikasi ini memungkinkan mahasiswa untuk mengelola rencana studi mereka dengan batasan seperti kapasitas kelas, maksimum SKS (24 SKS), dan validasi kelas yang sudah diambil. Implementasi ini mencerminkan sistem nyata seperti yang digunakan di universitas, dengan antarmuka berbasis teks yang sederhana namun fungsional.

Arsitektur Program

Program ini dirancang dengan pendekatan modular, di mana setiap kelas memiliki tanggung jawab spesifik. Mahasiswa mengelola data pribadi dan operasi FRS, Dosen menyimpan informasi pengajar, Matakuliah mendefinisikan mata kuliah, Kelas mengelola detail kelas dan daftar mahasiswa, serta DisplayFRS berfungsi sebagai pengelola data dan autentikasi. Kelas Main menyediakan antarmuka pengguna berbasis konsol dengan menu interaktif.

Implementasi Program

Aplikasi ini diimplementasikan menggunakan Java dengan BlueJ sebagai lingkungan pengembangan. Setiap kelas dirancang untuk memisahkan logika bisnis dari antarmuka pengguna, memastikan modularitas dan kemudahan pemeliharaan. Berikut adalah kode lengkap untuk masing-masing kelas beserta penjelasan:

Class Mahasiswa


import java.util.*;
class Mahasiswa {
    private String nama;
    private String nrp;
    private String password;
    private int sks = 0;
    private int maxSKS = 24;
    private List daftarKelas = new ArrayList<>();

    public Mahasiswa(String nama, String nrp, String password) {
        this.nama = nama;
        this.nrp = nrp;
        this.password = password;
    }

    public String getNrp() { return nrp; }
    public String getPassword() { return password; }
    public String getNama() { return nama; }

    public boolean ambilKelas(Kelas k) {
        if (hitungTotalSKS() + k.getMatakuliah().getSks() <= maxSKS && !k.sudahDiambil(this) && k.cekKapasitas()) {
            daftarKelas.add(k);
            k.tambahMahasiswa(this);
            sks = sks + k.getMatakuliah().getSks();
            return true;
        }
        return false;
    }

    public boolean dropKelas(Kelas k) {
        if (daftarKelas.remove(k)) {
            k.hapusMahasiswa(this);
            sks = sks - k.getMatakuliah().getSks();
            return true;
        }
        return false;
    }

    private int hitungTotalSKS() {
        return sks;
    }

    public void tampilkanFRS() {
        System.out.println("FRS " + nama + " (" + nrp + "):");
        for (Kelas k : daftarKelas) {
            System.out.println(" " + k.getKodeKelas() 
                               + "  " + k.getKodeRuangan() 
                               + " " + k.getMatakuliah().getNama()
                               + " " + k.getKelas()
                               + " (" + k.getMatakuliah().getSks() + " SKS)"
                               + " [" + k.getJadwal() + "]");
        }
        System.out.println("Total SKS: " + hitungTotalSKS());
    }
}

Class Kelas


import java.util.*;
class Kelas {
    private String kodeKelas;
    private String kodeRuangan;
    private String kelas;
    private Dosen dosenPengampu;
    private String jadwal;
    private int kapasitas;
    private Matakuliah matakuliah;
    private List daftarMahasiswa = new ArrayList<>();

    public Kelas(String kodeKelas, String kodeRuangan, String kelas, Matakuliah matkul, Dosen dosen, String jadwal, int kapasitas) {
        this.kodeKelas = kodeKelas;
        this.kodeRuangan = kodeRuangan;
        this.kelas = kelas;
        this.matakuliah = matkul;
        this.dosenPengampu = dosen;
        this.jadwal = jadwal;
        this.kapasitas = kapasitas;
    }

    public boolean cekKapasitas() {
        return daftarMahasiswa.size() < kapasitas;
    }

    public void tambahMahasiswa(Mahasiswa m) {
        daftarMahasiswa.add(m);
    }
    
    public boolean sudahDiambil(Mahasiswa m) {
        return daftarMahasiswa.contains(m);
    }

    public void hapusMahasiswa(Mahasiswa m) {
        daftarMahasiswa.remove(m);
    }

    public String getKodeKelas() { return kodeKelas; }
    public String getKodeRuangan() { return kodeRuangan; }
    public String getKelas() { return kelas; }
    public Matakuliah getMatakuliah() { return matakuliah; }
    public String getJadwal() { return jadwal; }
    public Dosen getDosen() { return dosenPengampu; }
}

Class Main


import java.util.*;
public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        DisplayFRS app = new DisplayFRS();

        boolean jalanApp = true;
        while (jalanApp) {
            // ==== LOGIN PAGE ====
            Mahasiswa mhs = null;
            while (mhs == null) {
                System.out.print("\nMasukkan NRP (atau 'q' untuk keluar): ");
                String nrp = sc.nextLine();
                if (nrp.equalsIgnoreCase("q")) {
                    System.out.println("Keluar dari aplikasi...");
                    sc.close();
                    return; // keluar program
                }

                System.out.print("Masukkan Password: ");
                String pw = sc.nextLine();

                mhs = app.loginMahasiswa(nrp, pw);
                if (mhs == null) {
                    System.out.println("Login gagal! Silakan coba lagi.\n");
                }
            }

            // ==== MENU PAGE ====
            System.out.println("Selamat datang, " + mhs.getNama());
            boolean jalanMenu = true;

            while (jalanMenu) {
                System.out.println("\nMenu:");
                System.out.println("1. Lihat Daftar Kelas");
                System.out.println("2. Ambil Kelas");
                System.out.println("3. Drop Kelas");
                System.out.println("4. Lihat FRS");
                System.out.println("5. Logout");
                System.out.println("6. Keluar Aplikasi");
                System.out.print("Pilihan: ");
                int pilih = sc.nextInt();
                sc.nextLine();

                switch (pilih) {
                    case 1:
                        app.tampilkanDaftarKelas();
                        break;
                    case 2:
                        app.tampilkanDaftarKelas();
                        System.out.print("Masukkan kode Kelas yang ingin diambil: ");
                        String kodeAdd = sc.nextLine();
                        boolean foundAdd = false;
                        for (Kelas k : app.getDaftarKelas()) {
                            if (k.getKodeKelas().equalsIgnoreCase(kodeAdd)) {
                                foundAdd = true;
                                if (mhs.ambilKelas(k)) {
                                    System.out.println("Kelas berhasil diambil!");
                                } else {
                                    System.out.println("Gagal ambil kelas (kapasitas penuh / SKS tidak cukup / Kelas sudah diambil).");
                                }
                            }
                        }
                        if (!foundAdd) {
                            System.out.println("Kode kelas tidak ditemukan.");
                        }
                        break;
                    case 3:
                        System.out.print("Masukkan kode kelas yang ingin di-drop: ");
                        String kodeDrop = sc.nextLine();
                        boolean foundDrop = false;
                        for (Kelas k : app.getDaftarKelas()) {
                            if (k.getKodeKelas().equalsIgnoreCase(kodeDrop)) {
                                foundDrop = true;
                                if (mhs.dropKelas(k)) {
                                    System.out.println("Kelas berhasil di-drop!");
                                } else {
                                    System.out.println("Kelas tidak ada di FRS Anda.");
                                }
                            }
                        }
                        if (!foundDrop) {
                            System.out.println("Kode kelas tidak ditemukan.");
                        }
                        break;
                    case 4:
                        mhs.tampilkanFRS();
                        break;
                    case 5:
                        System.out.println("Logout berhasil.\n");
                        jalanMenu = false; // kembali ke login page
                        break;
                    case 6:
                        System.out.println("Keluar dari aplikasi...");
                        jalanApp = false; // keluar dari aplikasi utama
                        jalanMenu = false;
                        break;
                    default:
                        System.out.println("Pilihan tidak valid.");
                }
            }
        }

        sc.close();
    }
}

Penjelasan Konsep

Aplikasi FRS ini dirancang untuk mendemonstrasikan konsep-konsep inti Pemrograman Berbasis Objek, termasuk:

  • Encapsulation: Atribut seperti nama, nrp, dan sks di kelas Mahasiswa bersifat privat, hanya dapat diakses melalui getter atau method spesifik, menjaga integritas data.
  • Composition: Kelas Kelas mengandung objek Matakuliah dan Dosen, mencerminkan hubungan "has-a" yang kuat.
  • Asosiasi: Hubungan dua arah antara Mahasiswa dan Kelas diimplementasikan melalui ArrayList, memungkinkan mahasiswa mengambil banyak kelas dan kelas memiliki banyak mahasiswa.
  • Modularitas: Setiap kelas memiliki tanggung jawab spesifik, memudahkan pemeliharaan dan pengembangan lebih lanjut.
  • Penggunaan Struktur Data: ArrayList digunakan untuk mengelola daftar mahasiswa, kelas, dosen, dan mata kuliah, menunjukkan fleksibilitas dalam pengelolaan koleksi.

Detail Implementasi Kelas

Class Mahasiswa

Kelas Mahasiswa adalah inti dari aplikasi, mengelola data pribadi mahasiswa dan operasi FRS. Atribut seperti sks dan daftarKelas diperbarui secara dinamis saat mahasiswa mengambil atau menghapus kelas.

  • Constructor Mahasiswa(String nama, String nrp, String password) → Menginisialisasi objek dengan nama, NRP, dan password. sks diatur ke 0, dan daftarKelas sebagai ArrayList kosong.
  • Method ambilKelas(Kelas k) → Memeriksa tiga kondisi sebelum menambahkan kelas: total SKS tidak melebihi maxSKS (24), kelas belum diambil (sudahDiambil()), dan kapasitas kelas tersedia (cekKapasitas()). Jika valid, kelas ditambahkan ke daftarKelas, SKS diperbarui, dan mahasiswa ditambahkan ke kelas.
  • Method dropKelas(Kelas k) → Menghapus kelas dari daftarKelas, mengurangi sks, dan menghapus mahasiswa dari daftar kelas. Mengembalikan true jika berhasil.
  • Method hitungTotalSKS() → Mengembalikan nilai sks, yang selalu diperbarui saat kelas ditambahkan atau dihapus.
  • Method tampilkanFRS() → Menampilkan daftar kelas dalam format terstruktur, termasuk kode kelas, ruangan, nama mata kuliah, kelas, SKS, dan jadwal, diikuti dengan total SKS.
  • Getter methods → getNrp(), getPassword(), dan getNama() menyediakan akses aman ke atribut privat.
Class Dosen

Kelas Dosen adalah kelas sederhana yang menyimpan informasi pengajar, digunakan dalam kelas Kelas untuk menunjukkan dosen pengampu.

  • Constructor Dosen(String nama, String nip) → Menginisialisasi dosen dengan nama dan NIP.
  • Method getNama() → Mengembalikan nama dosen untuk ditampilkan dalam daftar kelas atau FRS.
Class Kelas

Kelas Kelas mengelola informasi kelas seperti kode, ruangan, jadwal, dan daftar mahasiswa. Menggunakan composition untuk menyimpan objek Matakuliah dan Dosen.

  • Constructor Kelas(String kodeKelas, String kodeRuangan, String kelas, Matakuliah matkul, Dosen dosen, String jadwal, int kapasitas) → Menginisialisasi kelas dengan semua atribut dan daftarMahasiswa sebagai ArrayList kosong.
  • Method cekKapasitas() → Mengembalikan true jika jumlah mahasiswa kurang dari kapasitas maksimum.
  • Method tambahMahasiswa(Mahasiswa m), hapusMahasiswa(Mahasiswa m) → Mengelola daftar mahasiswa yang mengambil kelas.
  • Method sudahDiambil(Mahasiswa m) → Memeriksa apakah mahasiswa sudah terdaftar di kelas menggunakan contains().
  • Getter methods → Mengembalikan atribut seperti kodeKelas, matakuliah, dosen, jadwal, dll.
Class Matakuliah

Kelas Matakuliah menyimpan informasi dasar mata kuliah, seperti nama dan jumlah SKS, yang digunakan oleh kelas Kelas.

  • Constructor Matakuliah(String nama, int sks) → Menginisialisasi mata kuliah dengan nama dan SKS.
  • Method getNama(), getSks() → Getter untuk mengakses atribut privat.
Class DisplayFRS

Kelas DisplayFRS bertindak sebagai pengelola data utama, menyimpan daftar mahasiswa, dosen, mata kuliah, dan kelas, serta menyediakan fungsi login dan tampilan daftar kelas.

  • Constructor DisplayFRS() → Menginisialisasi data awal untuk 20 mahasiswa, 5 dosen, 10 mata kuliah, dan 15 kelas, yang mencerminkan data realistis untuk simulasi akademik.
  • Method loginMahasiswa(String nrp, String password) → Memverifikasi NRP dan password dengan mencocokkan terhadap daftarMahasiswa. Mengembalikan objek Mahasiswa jika valid, atau null jika gagal.
  • Method tampilkanDaftarKelas() → Menampilkan semua kelas dalam format yang informatif, termasuk kode, nama mata kuliah, kelas, SKS, dosen, ruangan, dan jadwal.
  • Method getDaftarKelas() → Mengembalikan daftarKelas untuk digunakan dalam operasi pengambilan atau penghapusan kelas.
Class Main

Kelas Main menyediakan antarmuka konsol interaktif untuk menjalankan aplikasi FRS, menggunakan Scanner untuk input pengguna dan struktur while serta switch untuk logika menu.

  • Method main(String[] args) → Entry point program. Menginisialisasi DisplayFRS dan menjalankan dua loop utama: satu untuk login (jalanApp) dan satu untuk menu (jalanMenu). Menu mendukung enam opsi: lihat daftar kelas, ambil kelas, drop kelas, lihat FRS, logout, atau keluar aplikasi.

Alur Kerja Aplikasi

Berikut adalah alur kerja utama aplikasi FRS:

  1. Inisialisasi: Kelas DisplayFRS mengisi data awal untuk mahasiswa, dosen, mata kuliah, dan kelas.
  2. Login: Pengguna memasukkan NRP dan password. Sistem memanggil loginMahasiswa() untuk memverifikasi dan mengembalikan objek Mahasiswa.
  3. Menu: Setelah login, pengguna memilih opsi melalui antarmuka konsol. Opsi seperti ambil kelas atau drop kelas memeriksa validasi (SKS, kapasitas, dll.) sebelum memperbarui state.
  4. Pengelolaan FRS: Method ambilKelas() dan dropKelas() memastikan konsistensi data antara Mahasiswa dan Kelas.
  5. Tampilan FRS: Method tampilkanFRS() memberikan ringkasan kelas yang diambil dan total SKS.
  6. Logout/Keluar: Pengguna dapat logout untuk kembali ke login atau keluar dari aplikasi sepenuhnya.

Kesimpulan

Tugas ini berhasil mengimplementasikan sistem FRS yang fungsional dengan pendekatan PBO. Konsep encapsulation memastikan data terlindungi, composition memungkinkan hubungan antar kelas yang kuat, dan penggunaan ArrayList mendukung pengelolaan data dinamis. Aplikasi ini mencerminkan sistem akademik nyata dengan batasan seperti SKS maksimum dan kapasitas kelas. Ke depannya, aplikasi ini dapat ditingkatkan dengan menambahkan antarmuka grafis (GUI) menggunakan Swing atau integrasi database untuk penyimpanan data permanen, memperluas fungsionalitas ke arah sistem informasi akademik yang lebih komprehensif.
