Tugas 8: Online Shop dengan Testing Menggunakan Java

3 November 2025
Waktu pengerjaan: 1 Hari

Deskripsi Tugas

Pada praktikum kali ini, saya mengembangkan aplikasi online shop sederhana menggunakan Java dengan menerapkan konsep Test-Driven Development (TDD). Project ini bertujuan untuk memahami pentingnya testing dalam siklus pengembangan software dan bagaimana cara mengimplementasikan unit testing menggunakan framework JUnit.

Testing merupakan fase krusial dalam software development lifecycle yang tidak boleh diabaikan. Aktivitas testing dilakukan untuk mengidentifikasi dan mendeteksi sebanyak mungkin kesalahan, bug, error, maupun defect sebelum aplikasi di-deploy ke production. Proses testing dilakukan dengan mengacu pada test plan dan test cases yang telah dirancang sebelumnya. Dalam praktik profesional, idealnya testing dilakukan oleh quality assurance (QA) tester yang independen dari tim programmer untuk menjaga objektivitas hasil testing dan menghindari bias.

Debugging adalah proses lanjutan yang dilakukan setelah testing mengidentifikasi adanya bug atau error. Aktivitas debugging meliputi pencarian lokasi spesifik dari kesalahan dan melakukan perbaikan terhadap code yang bermasalah. Umumnya, debugging menjadi tanggung jawab programmer yang menulis kode tersebut. Proses debugging dapat sangat time-consuming dan challenging, sehingga diperlukan strategi yang efektif. Terdapat tiga pendekatan utama dalam debugging:

  • Brute Force: Metode ini menggunakan pendekatan trial-and-error dengan menambahkan statement print atau logging di berbagai bagian kode untuk melacak flow eksekusi dan nilai variable. Meskipun sederhana, metode ini kurang efisien untuk kode yang kompleks.
  • Backtracking: Pendekatan ini melibatkan analisis source code secara sistematis untuk mengidentifikasi kemungkinan root cause dari error. Developer bergerak mundur dari titik error hingga menemukan lokasi awal munculnya masalah. Metode ini lebih efisien dibanding brute force karena lebih terstruktur.
  • Cause Elimination: Teknik ini menggunakan pendekatan binary search atau divide-and-conquer dengan membagi program menjadi partisi-partisi dan secara bertahap mempersempit area pencarian hingga lokasi error ditemukan. Ini adalah metode yang paling efisien untuk aplikasi besar.

Class Comment

public class Comment {
    private String author;
    private String text;
    private int rating;
    private int upvotes;
    private int downvotes;
    
    public Comment(String author, String text, int rating) {
        this.author = author;
        this.text = text;
        this.rating = rating;
        this.upvotes = 0;
        this.downvotes = 0;
    }
    
    public String getAuthor() {
        return author;
    }
    
    public String getText() {
        return text;
    }
    
    public int getRating() {
        return rating;
    }
    
    public int getUpvotes() {
        return upvotes;
    }
    
    public int getDownvotes() {
        return downvotes;
    }

    public int getVoteCount() {
        return upvotes - downvotes;
    }

    public void upvote() {
        upvotes++;
    }

    public void downvote() {
        downvotes++;
    }
    
    public String getFullDetails() {
        return "Author: " + author + "\nRating: " + rating + "\nComment: " + text + "\nUpvotes: " + upvotes + "\nDownvotes: " + downvotes;
    }
}

Penjelasan Implementasi:
Class Comment merepresentasikan sebuah review atau ulasan yang diberikan oleh customer terhadap produk dalam online shop. Struktur class ini dirancang dengan five private attributes untuk maintaining data integrity: author untuk identitas reviewer, text untuk konten ulasan, rating untuk nilai bintang produk, serta upvotes dan downvotes untuk tracking sistem voting helpfulness dari review tersebut.

Constructor class menerima tiga parameters (author, text, rating) dan secara otomatis menginisialisasi vote counters ke nilai nol. Class ini menyediakan complete set of getter methods untuk accessing semua attributes, yang sejalan dengan prinsip encapsulation. Method getVoteCount() melakukan kalkulasi net votes dengan formula (upvotes - downvotes) untuk determining overall helpfulness rating. Methods upvote() dan downvote() berfungsi sebagai mutators untuk increment respective vote counters. Method getFullDetails() menghasilkan formatted string representation dari entire comment data yang ready untuk display purposes.

Class SalesItem

import java.util.ArrayList;
import java.util.Iterator;

public class SalesItem {
    private String name;
    private int price;
    private ArrayList comments;
    
    public SalesItem(String name, int price) {
        this.name = name;
        this.price = price;
        comments = new ArrayList();
    }
    
    public String getName() {
        return name;
    }
    
    public int getPrice() {
        return price;
    }
    
    public int getNumberOfComments() {
        return comments.size();
    }
    
    public boolean addComment(String author, String text, int rating) {
        if (ratingInvalid(rating)) return false;
        if (findCommentByAuthor(author) != null) return false;
        comments.add(new Comment(author, text, rating));
        return true;
    }
    
    public void removeComment(int index) {
        if (index >= 0 && index < comments.size()) comments.remove(index);
    }
    
    public void upvoteComment(int index) {
        if (index >= 0 && index < comments.size()) comments.get(index).upvote();
    }
    
    public void downvoteComment(int index) {
        if (index >= 0 && index < comments.size()) comments.get(index).downvote();
    }
    
    public void showInfo() {
        System.out.println("*** " + name + " ***");
        System.out.println("Price: " + priceString(price));
        System.out.println();
        System.out.println("Customer comments:");
        for (Comment comment : comments) {
            System.out.println("-----------------------------------");
            System.out.println(comment.getFullDetails());
        }
        System.out.println();
        System.out.println("=====================================");
    }
    
    public Comment findMostHelpfulComment() {
        Iterator it = comments.iterator();
        Comment best = it.next();
        while (it.hasNext()) {
            Comment current = it.next();
            if (current.getVoteCount() > best.getVoteCount()) {
                best = current;
            }
        }
        return best;
    }
    
    private boolean ratingInvalid(int rating) {
        return rating < 1 || rating > 5;
    }
    
    public Comment findCommentByAuthor(String author) {
        for (Comment comment : comments) {
            if (comment.getAuthor().equals(author)) {
                return comment;
            }
        }
        return null;
    }
    
    public Comment findCommentByText (String text) {
        for (Comment comment : comments) {
            if (comment.getText().equals(text)) {
                return comment;
            }
        }
        return null;
    }
    
    private String priceString(int price) {
        int dollars = price / 100;
        int cents = price - (dollars * 100);
        if (cents <= 9) return "$" + dollars + ".0" + cents;
        else return "$" + dollars + "." + cents;
    }
}

Penjelasan Implementasi:
Class SalesItem adalah core entity yang merepresentasikan produk dalam sistem online shop. Class ini memiliki three key attributes: name untuk product identifier, price yang stored dalam cents format untuk precision, dan comments sebagai ArrayList untuk managing collection dari customer reviews.

Constructor melakukan initialization dari product name dan price, serta instantiate empty ArrayList untuk comments storage. Method addComment() implements comprehensive validation logic: pertama checking rating validity menggunakan ratingInvalid() helper method yang ensures rating berada dalam range 1-5, kemudian verifying bahwa author belum pernah submit review untuk mencegah duplicate reviews dari same user. Validation ini critical untuk maintaining data integrity dan review quality.

Methods removeComment(), upvoteComment(), dan downvoteComment() semuanya implement bounds checking untuk preventing index out of bounds exceptions. Method showInfo() generates formatted display dari product information beserta all associated reviews dengan clean visual separators. Method findMostHelpfulComment() utilizes Iterator pattern untuk traverse comments collection dan determine comment dengan highest vote count menggunakan greedy algorithm approach.

Helper methods findCommentByAuthor() dan findCommentByText() implement linear search algorithm untuk comment retrieval berdasarkan specific criteria. Private method priceString() melakukan price formatting dengan converting cents-based integer ke currency string format dengan proper decimal representation.

Class SalesItemTest

import static org.junit.Assert.*;
import org.junit.After;
import org.junit.Before;
import org.junit.Test;

public class SalesItemTest
{
    public SalesItemTest(){}

    @Before
    public void setUp() {}
    
    @After
    public void tearDown() {}
    
    @Test
    public void testAddComment()
    {
        SalesItem salesIte1 = new SalesItem("Item1", 12999);
        assertEquals(true, salesIte1.addComment("Felix", "Absolutely loved it", 5));
        assertEquals(1, salesIte1.getNumberOfComments());
    }

    @Test
    public void testIllegalRating()
    {
        SalesItem salesIte1 = new SalesItem("Item1", 12999);
        assertEquals(false, salesIte1.addComment("BadUser", "Terrible product", 0));
    }

    @Test
    public void testDuplicateAuthor()
    {
        SalesItem salesIte1 = new SalesItem("Item1", 12999);
        assertEquals(true, salesIte1.addComment("Felix", "Absolutely loved it", 5));
        assertEquals(false, salesIte1.addComment("Felix", "Could be better", 3));
    }

    @Test
    public void testInit()
    {
        SalesItem salesIte1 = new SalesItem("Sample Item", 1500);
        assertEquals("Sample Item", salesIte1.getName());
        assertEquals(1500, salesIte1.getPrice());
    }

    @Test
    public void addComment()
    {
        SalesItem salesIte1 = new SalesItem("Advanced Gardening Guide", 4999);
        assertEquals(true, salesIte1.addComment("Sam", "Easy to follow, my plants thrive!", 4));
    }
}

Penjelasan Implementasi:
Class SalesItemTest adalah unit test suite yang developed menggunakan JUnit framework untuk verifying functionality dan correctness dari SalesItem class. Test class ini follows standard JUnit conventions dengan utilizing annotations untuk test lifecycle management.

Method setUp() dengan annotation @Before dan tearDown() dengan @After disediakan sebagai hooks untuk test preparation dan cleanup, meskipun dalam implementation ini tidak perform specific actions. Each test method marked dengan @Test annotation represents independent test case yang focuses pada specific functionality aspect.

Test testAddComment() verifies basic comment addition functionality dengan asserting return value adalah true dan comment count increases correctly. Test testIllegalRating() validates input validation logic dengan attempting to add comment dengan invalid rating (0) dan expecting rejection. Test testDuplicateAuthor() ensures duplicate prevention mechanism bekerja dengan attempting second comment dari same author dan expecting failure. Test testInit() validates constructor behavior dengan checking proper initialization dari name dan price attributes. Additional addComment() test provides another successful addition scenario dengan different test data.

Test suite ini demonstrates important testing principles: test isolation (each test independent), clear assertions (using assertEquals), dan comprehensive coverage dari different scenarios including normal cases, boundary conditions, dan error cases.

Analisis Konsep Testing

Implementation online shop project ini mendemonstrasikan several important software testing concepts dan best practices:

  • Test-Driven Development (TDD): Project ini mengikuti TDD approach dimana test cases ditulis untuk define expected behavior sebelum atau bersamaan dengan implementation code. Ini ensures bahwa code developed dengan clear specifications dan automatically testable dari awal development process.
  • Unit Testing: Setiap test method focuses pada testing single unit of functionality secara isolated. Misalnya, testAddComment() specifically tests comment addition feature tanpa dependencies pada other system components, making tests more reliable dan easier to debug ketika failures occur.
  • Input Validation Testing: Test cases seperti testIllegalRating() dan testDuplicateAuthor() verify bahwa system properly handles invalid inputs dan edge cases. Ini critical untuk preventing bugs dan ensuring robust application behavior dalam real-world usage scenarios.
  • Assertion-Based Verification: Penggunaan assertEquals() statements provides clear verification points yang automatically checked oleh JUnit framework. Ini eliminates manual verification dan enables automated regression testing untuk quickly detect breakages ketika code changes.
  • Test Coverage: Test suite covers multiple aspects dari SalesItem functionality termasuk initialization, comment addition dengan various scenarios, dan validation logic. Comprehensive test coverage increases confidence dalam code correctness dan reduces likelihood dari undetected bugs.

Kesimpulan

Penerapan Test-Driven Development methodology dalam pengembangan aplikasi online shop ini successfully demonstrates nilai dan importance dari systematic testing approach. Dengan menulis test cases yang comprehensive, kita dapat ensure bahwa setiap functionality component telah thoroughly validated sebelum integration ke larger system.

TDD approach memberikan multiple benefits: meningkatkan code quality melalui continuous verification, mengurangi bug count dengan early detection, memfasilitasi safe refactoring dengan regression test safety net, dan menghasilkan living documentation dalam bentuk executable test specifications. Implementation testing yang proper adalah fundamental practice dalam professional software development yang directly contributes ke reliability, maintainability, dan overall quality dari software products.
