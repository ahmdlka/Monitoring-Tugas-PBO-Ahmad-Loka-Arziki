Tugas Aplikasi CRUD Perpustakaan dengan Database MySQL

9 December 2025
Waktu pengerjaan: 2 hari
Java MySQL JDBC CRUD Database PBO

Pengantar

Dalam pengembangan aplikasi modern, kemampuan untuk mengelola data secara efektif merupakan aspek fundamental yang harus dikuasai. CRUD (Create, Read, Update, Delete) adalah empat operasi dasar dalam manajemen data yang menjadi fondasi hampir semua sistem informasi. Project ini mengimplementasikan aplikasi manajemen perpustakaan sederhana yang mendemonstrasikan integrasi antara aplikasi Java dengan database MySQL menggunakan JDBC (Java Database Connectivity).

Aplikasi ini dibangun menggunakan IDE BlueJ dan menyediakan antarmuka console-based untuk melakukan operasi CRUD pada data buku perpustakaan. Melalui project ini, saya mempelajari bagaimana Java berkomunikasi dengan database relasional, menjalankan query SQL, dan mengelola koneksi database dengan aman dan efisien.

Sistem ini memungkinkan pengguna untuk menambahkan buku baru ke database, melihat daftar semua buku yang tersedia, memperbarui informasi buku yang sudah ada, serta menghapus buku dari koleksi perpustakaan. Setiap operasi diimplementasikan dengan memanfaatkan API JDBC yang menyediakan abstraksi standar untuk akses database di Java.

1. Persiapan Database

Langkah pertama dalam membangun aplikasi adalah menyiapkan infrastruktur database. Kita akan membuat database bernama perpustakaan dengan tabel buku yang memiliki struktur kolom untuk menyimpan informasi dasar buku.

Skrip SQL untuk Setup Database:

CREATE DATABASE perpustakaan_pbo;

USE perpustakaan_pbo;

CREATE TABLE buku (
    id_buku INT NOT NULL AUTO_INCREMENT,
    judul VARCHAR(45) NOT NULL,
    pengarang VARCHAR(45) NOT NULL,
    PRIMARY KEY (id_buku)
);

INSERT INTO buku (judul, pengarang) VALUES 
('Belajar Pemrograman Java dari Nol', 'Petani Kode'),
('Pemrograman Java Menggunakan Linux', 'Petani Kode');

Penjelasan Setup Database:

Perintah CREATE DATABASE perpustakaan membuat database baru yang akan menjadi container untuk semua data aplikasi. Perintah USE perpustakaan mengaktifkan database tersebut untuk operasi selanjutnya.

Tabel buku dirancang dengan tiga kolom: id_buku sebagai primary key dengan auto-increment (database akan otomatis generate ID unik), judul untuk menyimpan nama buku dengan maksimal 45 karakter, dan pengarang untuk nama penulis. Constraint NOT NULL memastikan kolom-kolom penting harus diisi.

Data sampel diinisialisasi dengan perintah INSERT INTO untuk memberikan starting point saat menguji aplikasi. Ini memudahkan testing fitur read, update, dan delete tanpa harus menambahkan data manual terlebih dahulu.

2. Konfigurasi Driver MySQL di BlueJ

BlueJ tidak menyertakan MySQL Connector secara default dalam library-nya, sehingga kita perlu menambahkan driver JDBC secara manual. Driver ini berfungsi sebagai jembatan komunikasi antara aplikasi Java dengan MySQL server.

Langkah-langkah Instalasi Driver:

  • Download MySQL Connector/J (file JAR) dari website resmi MySQL atau Maven Repository. Pastikan versi yang didownload kompatibel dengan versi MySQL server yang terinstall.
  • Buka BlueJ, kemudian navigasi ke menu Tools → Preferences → Libraries.
  • Klik tombol Add dan pilih file JAR MySQL Connector yang telah didownload.
  • Restart BlueJ agar library baru ter-load dengan benar ke dalam classpath.
  • Verifikasi instalasi dengan mencoba import com.mysql.jdbc.Driver atau com.mysql.cj.jdbc.Driver (untuk versi terbaru) dalam kode Java.

Tanpa driver ini, kode Java tidak akan bisa melakukan koneksi ke database MySQL dan akan menghasilkan ClassNotFoundException saat runtime.

3. Implementasi Kode Java

Setelah database dan driver siap, kita akan mengimplementasikan logika aplikasi dalam kelas JavaCRUD. Kelas ini mengintegrasikan semua komponen: koneksi database, user interface console, dan operasi CRUD.

Class JavaCRUD.java

import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.sql.DriverManager;
import java.sql.Connection;
import java.sql.Statement;
import java.sql.ResultSet;

public class JavaCRUD {
    static final String JDBC_DRIVER = "com.mysql.jdbc.Driver";
    static final String DB_URL = "jdbc:mysql://localhost/perpustakaan_pbo";
    static final String USER = "root";
    static final String PASS = ""; // Sesuaikan password di databasemu
    static Connection conn;
    static Statement stmt;
    static ResultSet rs;
    static InputStreamReader inputStreamReader = new InputStreamReader(System.in);
    static BufferedReader input = new BufferedReader(inputStreamReader);

    public static void main(String[] args) {
        try {
            Class.forName(JDBC_DRIVER);

            conn = DriverManager.getConnection(DB_URL, USER, PASS);
            stmt = conn.createStatement();

            while (!conn.isClosed()) {
                showMenu();
            }

            stmt.close();
            conn.close();

        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    static void showMenu() {
        System.out.println("\n========= MENU UTAMA =========");
        System.out.println("1. Insert Data");
        System.out.println("2. Show Data");
        System.out.println("3. Edit Data");
        System.out.println("4. Delete Data");
        System.out.println("0. Keluar");
        System.out.println("");
        System.out.print("PILIHAN> ");

        try {
            int pilihan = Integer.parseInt(input.readLine());

            switch (pilihan) {
                case 0:
                    System.exit(0);
                    break;
                case 1:
                    insertBuku();
                    break;
                case 2:
                    showData();
                    break;
                case 3:
                    updateBuku();
                    break;
                case 4:
                    deleteBuku();
                    break;
                default:
                    System.out.println("Pilihan salah!");
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    static void showData() {
        String sql = "SELECT * FROM buku";
        try {
            rs = stmt.executeQuery(sql);
            
            System.out.println("+--------------------------------+");
            System.out.println("|    DATA BUKU DI PERPUSTAKAAN   |");
            System.out.println("+--------------------------------+");

            while (rs.next()) {
                int idBuku = rs.getInt("id_buku");
                String judul = rs.getString("judul");
                String pengarang = rs.getString("pengarang");

                System.out.println(String.format("%d. %s -- (%s)", idBuku, judul, pengarang));
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    static void insertBuku() {
        try {
            System.out.print("Judul: ");
            String judul = input.readLine().trim();
            System.out.print("Pengarang: ");
            String pengarang = input.readLine().trim();

            String sql = "INSERT INTO buku (judul, pengarang) VALUES('%s', '%s')";
            sql = String.format(sql, judul, pengarang);

            stmt.execute(sql);
            System.out.println("Data berhasil disimpan...");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    static void updateBuku() {
        try {
            System.out.print("ID yang mau diedit: ");
            int idBuku = Integer.parseInt(input.readLine());
            System.out.print("Judul: ");
            String judul = input.readLine().trim();
            System.out.print("Pengarang: ");
            String pengarang = input.readLine().trim();

            String sql = "UPDATE buku SET judul='%s', pengarang='%s' WHERE id_buku=%d";
            sql = String.format(sql, judul, pengarang, idBuku);

            stmt.execute(sql);
            System.out.println("Data berhasil diubah...");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    static void deleteBuku() {
        try {
            System.out.print("ID yang mau dihapus: ");
            int idBuku = Integer.parseInt(input.readLine());

            String sql = String.format("DELETE FROM buku WHERE id_buku=%d", idBuku);

            stmt.execute(sql);
            System.out.println("Data telah terhapus...");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

Penjelasan Detail Implementasi

Class JavaCRUD

Kelas JavaCRUD adalah core component yang mengenkapsulasi seluruh business logic aplikasi perpustakaan. Kelas ini mendefinisikan konstanta-konstanta penting untuk koneksi database:

  • JDBC_DRIVER: String identifier untuk MySQL JDBC driver class yang akan di-load oleh JVM.
  • DB_URL: Connection string dengan format jdbc:mysql://host:port/database_name. Localhost menunjukkan database server berjalan di mesin yang sama.
  • USER dan PASS: Kredensial autentikasi untuk akses database. Pada production, gunakan environment variables atau encrypted configuration.

Objek JDBC (Connection, Statement, ResultSet) dideklarasikan sebagai static member agar dapat diakses oleh semua method dalam kelas tanpa perlu passing parameter berulang kali. BufferedReader digunakan untuk membaca input user dari console secara efisien.

Method main()

Entry point aplikasi yang bertanggung jawab untuk lifecycle management:

  • Class.forName(JDBC_DRIVER) secara eksplisit memuat MySQL driver class ke dalam memory. Ini adalah requirement untuk JDBC driver registration di versi Java yang lebih lama.
  • DriverManager.getConnection() membuat koneksi aktif ke database server menggunakan kredensial yang diberikan. Koneksi ini akan dipertahankan selama aplikasi berjalan.
  • stmt = conn.createStatement() membuat Statement object yang akan digunakan untuk mengeksekusi SQL commands.
  • Loop while (!conn.isClosed()) menjaga aplikasi tetap running dan terus menampilkan menu hingga user memilih exit.
  • Cleanup resources dengan close() dipanggil pada akhir program untuk mencegah resource leaks, meskipun dalam practice sebenarnya loop infinite akan menghalangi eksekusi baris ini.

Method showMenu()

User interface console yang menyajikan menu navigasi aplikasi. Method ini:

  • Menampilkan daftar operasi yang tersedia dengan numbering yang clear (1-4 untuk CRUD, 0 untuk exit).
  • Membaca input user menggunakan BufferedReader.readLine() dan mengkonversi ke integer dengan Integer.parseInt().
  • Menggunakan switch-case statement untuk routing ke method yang sesuai berdasarkan pilihan user. Pattern ini memberikan clean separation of concerns.
  • Menangani invalid input dengan default case yang memberikan feedback error kepada user.
  • Exception handling memastikan aplikasi tidak crash saat user memasukkan input non-numeric.

Method showData()

Implementasi operasi Read dalam CRUD:

  • Menggunakan query SELECT * FROM buku untuk retrieve semua records dari tabel tanpa filtering.
  • stmt.executeQuery() menjalankan SELECT statement dan mengembalikan ResultSet yang berisi hasil query.
  • ResultSet di-iterate dengan while (rs.next()). Cursor awalnya berada sebelum row pertama, next() memindahkan cursor dan return true jika ada data.
  • Getter methods seperti getInt() dan getString() digunakan untuk extract data dari current row berdasarkan column name.
  • Data diformat dengan String.format() untuk tampilan yang rapi dan terstruktur di console.

Method insertBuku()

Implementasi operasi Create dalam CRUD:

  • Meminta input judul dan pengarang dari user melalui console prompt.
  • trim() digunakan untuk membersihkan whitespace di awal dan akhir string, mencegah data yang tidak rapi.
  • SQL INSERT statement dikonstruksi menggunakan String.format() untuk inject user input ke dalam query template. Note: Ini vulnerable terhadap SQL injection; production code harus menggunakan PreparedStatement.
  • stmt.execute() menjalankan INSERT command. Database akan auto-generate id_buku berkat AUTO_INCREMENT.
  • Feedback message diberikan kepada user untuk konfirmasi bahwa operasi berhasil.

Method updateBuku()

Implementasi operasi Update dalam CRUD:

  • Meminta user untuk specify ID buku yang akan di-update, memastikan operasi dilakukan pada record yang tepat.
  • User diminta memasukkan data baru untuk judul dan pengarang yang akan menggantikan nilai existing.
  • SQL UPDATE statement dengan WHERE clause memastikan hanya record dengan id_buku yang matching akan dimodifikasi.
  • Tanpa WHERE clause, semua records akan ter-update (dangerous operation).
  • Operasi ini tidak memvalidasi apakah ID exist; jika ID tidak ditemukan, query akan execute tapi tidak ada row yang affected.

Method deleteBuku()

Implementasi operasi Delete dalam CRUD:

  • User memasukkan ID buku yang akan dihapus dari database.
  • SQL DELETE statement dengan WHERE clause menghapus record specific berdasarkan primary key.
  • Operasi ini bersifat permanent dan irreversible. Production application sebaiknya implement soft delete atau confirmation dialog sebelum actual deletion.
  • stmt.execute() menjalankan deletion dan mengembalikan boolean indicating success/failure.
  • Seperti update, tidak ada validasi apakah ID exist before deletion attempt.

Konsep Pemrograman yang Diterapkan

  • JDBC (Java Database Connectivity): API standar Java untuk interaksi dengan database relasional. Menyediakan abstraksi yang memungkinkan aplikasi bekerja dengan berbagai database tanpa perubahan kode signifikan.
  • Database Connection Management: Membuat, mempertahankan, dan menutup koneksi database dengan proper resource handling untuk mencegah connection leaks.
  • SQL Query Execution: Mengeksekusi berbagai jenis SQL statements (SELECT, INSERT, UPDATE, DELETE) melalui JDBC Statement interface.
  • ResultSet Processing: Iterasi dan ekstraksi data dari query results menggunakan cursor-based navigation.
  • Exception Handling: Try-catch blocks untuk menangani SQLException dan IOException yang mungkin terjadi selama database operations atau user input.
  • Input/Output Streams: Menggunakan BufferedReader untuk efficient reading dari System.in untuk user interaction.
  • String Formatting: Konstruksi SQL queries dan output formatting menggunakan String.format() untuk readability.
  • Static Methods: Semua business logic diimplementasikan sebagai static methods untuk simplicity dalam console application context.
  • Control Flow: Switch-case statement untuk menu navigation dan while loop untuk persistent application state.

Gambar 1: Menu Utama Aplikasi CRUD Perpustakaan

Tampilan menu utama aplikasi yang menampilkan 5 pilihan operasi: Insert Data (1), Show Data (2), Edit Data (3), Delete Data (4), dan Keluar (0). Interface yang sederhana dan intuitif memudahkan pengguna untuk memilih operasi yang ingin dilakukan.

Proses update data buku existing. Pengguna memasukkan ID buku yang ingin diubah, kemudian memasukkan judul dan pengarang baru. Aplikasi mengeksekusi query UPDATE dan menampilkan konfirmasi "Data berhasil diubah..." untuk memberitahu bahwa perubahan telah tersimpan di database.

Proses penghapusan buku dari database. Pengguna cukup memasukkan ID buku yang ingin dihapus, dan aplikasi akan mengeksekusi query DELETE. Pesan "Data telah terhapus..." ditampilkan sebagai konfirmasi bahwa record telah berhasil dihapus dari tabel buku.

Kesimpulan

Project aplikasi CRUD Perpustakaan ini berhasil mendemonstrasikan implementasi lengkap dari operasi database fundamental menggunakan Java dan MySQL. Melalui penggunaan JDBC API, aplikasi dapat melakukan Create, Read, Update, dan Delete operations dengan efektif terhadap data buku dalam database relasional.

Implementasi ini memberikan pemahaman praktis tentang bagaimana aplikasi Java berkomunikasi dengan database server, menjalankan SQL queries, memproses result sets, dan menang

Secara keseluruhan, aplikasi ini berhasil memenuhi tujuan utama sebagai contoh implementasi CRUD sederhana namun fungsional. Pengembangan lebih lanjut dapat dilakukan dengan mengganti Statement menjadi PreparedStatement untuk keamanan yang lebih baik, menambahkan validasi input, atau meningkatkan antarmuka aplikasi menggunakan GUI seperti JavaFX atau Swing. Dengan demikian, aplikasi ini dapat menjadi dasar yang baik untuk pengembangan sistem manajemen data yang lebih kompleks di masa depan.
