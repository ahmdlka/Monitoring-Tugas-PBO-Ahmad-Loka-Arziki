Tugas 2: Pengenalan BlueJ dan Implementasi TicketMachine

Waktu pengerjaan: 1 hari

Deskripsi Tugas

Pada tugas pertama mata kuliah Pemrograman Berbasis Objek, saya mempelajari konsep dasar class dan object. Tugas ini mengharuskan saya untuk membuat program sederhana dalam bahasa Java yang mendemonstrasikan pembuatan class, pembuatan object, serta penggunaan atribut dan method.

Implementasi Program

Saya membuat program sederhana untuk mendefinisikan class TicketMachine dengan atribut price, balance, dan total. Class ini juga memiliki method.... 

/**
* TicketMachine models a ticket machine that issues
* flat-fare tickets.
* The price of a ticket is specified via the constructor.
* Instances will check to ensure that a user only enters
* sensible amounts of money, and will only print a ticket
* if enough money has been input.
*/
public class TicketMachine{
    // The price of a ticket from this machine.
    private int price;
    // The amount of money entered by a customer so far.
    private int balance;
    // The total amount of money collected by this machine.
    private int total;
    /**
    * Create a machine that issues tickets of the given price.
    */
    public TicketMachine(int cost){
        price = cost;
        balance = 0;
        total = 0;
        }
    /**
    * Return the price of a ticket.
    */
    public int getPrice(){
        return price;
        }
    /**
    * Return the amount of money already inserted for the
    * next ticket.
    */
    public int getBalance(){
        return balance;
        }
    /**
    * Receive an amount of money in cents from a customer.
    * Check that the amount is sensible.
    */
    public void insertMoney(int amount){
        if(amount > 0) {
            balance = balance + amount;
        }
        else {
            System.out.println("Use a positive amount rather than: " + 
            amount);
        }
    }
    /**
    * Print a ticket if enough money has been inserted, and
    * reduce the current balance by the ticket price. Print
    * an error message if more money is required.
    */
    public void printTicket(){
        if(balance >= price) {
            // Simulate the printing of a ticket.
            System.out.println("##################");
            System.out.println("# The BlueJ Line");
            System.out.println("# Ticket");
            System.out.println("# " + price + " cents.");
            System.out.println("##################");
            System.out.println();
            
            // Update the total collected with the price.
            
            total = total + price;
            
            // Reduce the balance by the price.
            balance = balance - price;
        }
        else {
            System.out.println("You must insert at least: " + 
            (price - balance) + " cents.");
        }
    }
    /**
    * Return the money in the balance.
    * The balance is cleared.
    */
    public int refundBalance(){
        int amountToRefund;
        amountToRefund = balance;
        balance = 0;
        return amountToRefund;
        }
}

Penjelasan Konsep

Program TicketMachine ini dibuat untuk mendemonstrasikan konsep dasar class dan object dalam pemrograman berorientasi objek. class berfungsi sebagai cetakan yang berisi atribut dan method, sedangkan object merupakan instance dari class tersebut dengan nilai data nyata.

Pada class TicketMachine terdapat tiga atribut utama:

  • price → menyimpan harga tiket yang ditentukan saat object dibuat melalui constructor.
  • balance → menyimpan jumlah uang yang sudah dimasukkan oleh pengguna.
  • total → mencatat total uang yang sudah terkumpul dari semua transaksi.

Constructor TicketMachine(int cost) digunakan untuk menginisialisasi harga tiket dan mengatur nilai awal balance serta total menjadi 0. Dalam implementasi di BlueJ, saya membuat ticket baru dengan harga 500.

Method getPrice() berfungsi mengembalikan harga tiket, sementara getBalance() mengembalikan jumlah uang yang sudah dimasukkan. Saat diuji di BlueJ, getPrice() menampilkan nilai 500 sesuai dengan harga tiket, dan getBalance() menampilkan perubahan nilai balance setiap kali uang dimasukkan.

Method insertMoney(int amount) memungkinkan pengguna memasukkan uang. Program juga memeriksa apakah nilai yang dimasukkan positif, sehingga tidak bisa memasukkan jumlah negatif. Pada implementasi, saya memasukkan uang sebesar 1000, sehingga balance menjadi 1000.

Method printTicket() hanya mencetak tiket jika balance cukup atau lebih besar dari price. Jika syarat terpenuhi, tiket akan tercetak dengan menampilkan harga, kemudian nilai total bertambah sebesar price, dan balance dikurangi dengan price. Pada uji coba, setelah memasukkan 1000, saya mencetak tiket sekali. Hasilnya tiket berhasil dicetak, balance berkurang menjadi 500, dan total bertambah 500.

Method refundBalance() digunakan untuk mengembalikan sisa uang yang ada pada balance dan kemudian mengosongkan balance tersebut. Pada implementasi, setelah saya mencetak tiket sekali, saya memanggil refundBalance(). Hasilnya, sisa balance sebesar 500 dikembalikan, dan balance menjadi 0.

Dari keseluruhan implementasi di BlueJ, terlihat jelas bagaimana sebuah class mendefinisikan atribut dan perilaku, lalu object dari class tersebut dapat digunakan untuk menjalankan fungsi-fungsi nyata. Interaksi antara insertMoney(), printTicket(), dan refundBalance() menunjukkan bagaimana perubahan state pada object terjadi sesuai dengan aksi pengguna.

Kesimpulan

Tugas ini memberikan pemahaman dasar tentang class dan object dalam pemrograman berorientasi objek. Saya belajar bagaimana mendefinisikan class dengan atribut dan method, membuat object dari class, serta mengakses atribut dan method dari object. Konsep ini merupakan fondasi penting untuk mempelajari konsep-konsep PBO yang lebih advanced seperti inheritance, polymorphism, dan encapsulation.
